# ════════════════════════════════════════════════════════════════════
#
#  HOW TO RUN
#     ./sim -schd fcfs tests/input_sched_race.txt
#     ./sim -schd sjf  tests/input_sched_race.txt
#
#  WHAT THIS TESTS
#     This is the real mutual-exclusion / race test:  7 travelers start on
#     7 DIFFERENT nodes (1..7), so nobody fights at the start.  Every start
#     node is the SAME distance (weight 1) from the central node 0, so all
#     7 travelers arrive at node 0 AT THE SAME INSTANT and crash into each
#     other there.  Only one traveler may be inside a node at a time, so:
#
#        - exactly ONE is let into node 0,
#        - the other SIX are forced to BLOCK and queue
#          (look for the  "... waiting for node 0"  lines in the output),
#        - the parent then drains that queue one-by-one, in the order the
#          chosen algorithm dictates.
#
#     Because node 0 is NOT where anyone started, this proves the queue /
#     blocking logic works in the MIDDLE of the journey, not just at t=0.
#
#  THE COLLISION POINT is node 0.  After node 0 each traveler peels off to
#  its own private destination (8..14), so there is exactly one clean
#  contention point — easy to read in the log.
#
#  MAP  (traveler : start -> 0 -> dest : total path cost):
#        T0 : 1 -> 0 -> 8    cost 14   (longest  path)
#        T1 : 2 -> 0 -> 9    cost 12
#        T2 : 3 -> 0 -> 10   cost 10
#        T3 : 4 -> 0 -> 11   cost  8
#        T4 : 5 -> 0 -> 12   cost  6
#        T5 : 6 -> 0 -> 13   cost  4
#        T6 : 7 -> 0 -> 14   cost  2   (shortest path)
#
#  EXPECTED ADMISSION ORDER AT NODE 0
#        FCFS -> arrival order :  T0, T1, T2, T3, T4, T5, T6
#                (they line up in the order they reach node 0)
#        SJF  -> cheapest first:  T6, T5, T4, T3, T2, T1, T0
#                (the reverse — shortest total path jumps the queue)
#
#     Run both and compare the "entered node 0" lines: same race, opposite
#     service order.  In BOTH runs node 0 is never occupied by two
#     travelers at once — that is the mutual exclusion holding under load.
# ════════════════════════════════════════════════════════════════════
